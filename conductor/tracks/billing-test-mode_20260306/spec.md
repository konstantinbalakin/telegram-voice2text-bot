# Specification: Billing Test Mode

**Track ID:** billing-test-mode_20260306
**Type:** Feature
**Created:** 2026-03-06
**Status:** Draft

## Summary

Добавить фича-флаг `billing_test_mode` для промежуточного деплоя биллинга на прод — с полным интерфейсом оплаты (тестовый ключ YooKassa), но со старым приветствием и без списания минут.

## Context

Для заключения договора с платёжной системой нужно продемонстрировать работающий интерфейс оплаты на продакшн-контуре. При этом реальные пользователи не должны видеть информацию о платных опциях и не должны терять минуты, т.к. оплата через тестовый ключ невозможна.

## User Story

Как владелец бота, я хочу развернуть биллинг на продакшене в тестовом режиме, чтобы показать платёжной системе скриншоты работающего интерфейса оплаты, при этом не затрагивая текущий пользовательский опыт.

## Acceptance Criteria

- [ ] Новый флаг `BILLING_TEST_MODE=true` в конфиге
- [ ] При `billing_enabled=true` + `billing_test_mode=true`: приветствие /start — старое (без информации о тарифах)
- [ ] При `billing_enabled=true` + `billing_test_mode=true`: минуты при транскрипции не списываются и лимит не проверяется
- [ ] Команды /buy, /balance, /subscribe и платёжные хендлеры работают полноценно
- [ ] Меню бота содержит команду /buy
- [ ] Миграции и seed-данные накатываются как обычно
- [ ] После отключения `billing_test_mode=false` — всё работает как полный биллинг

## Dependencies

Зависит от существующего кода биллинга (billing-system, yookassa-native, billing-ux-polish треки — все завершены).

## Out of Scope

- Изменение логики оплаты
- Новые миграции БД
- Изменение seed-данных
