# Specification: Smart Limit Warnings — Suppress Scary Alerts When Extra Minutes Available

**Track ID:** smart-limit-warning_20260311
**Type:** Feature
**Created:** 2026-03-11
**Status:** Draft

## Summary

Сейчас при исчерпании дневного лимита (или при достижении 80% порога) пользователь получает пугающее предупреждение вне зависимости от того, есть ли у него пакетные или бонусные минуты. Если у пользователя есть дополнительные минуты, предупреждение должно быть информационным (не пугающим), а при их отсутствии — оставаться текущим.

## Context

В биллинговой системе есть три источника минут:
1. **daily_limit** — бесплатные дневные минуты (сбрасываются каждый день)
2. **bonus_minutes** — бонусные минуты (welcome bonus и т.п.)
3. **package_minutes** — купленные пакеты минут

После завершения транскрипции в `transcription_orchestrator.py` вызывается `get_limit_status()`, и при статусах `"exhausted"` / `"warning"` пользователю отправляется предупреждение + экран баланса. Это поведение не учитывает наличие дополнительных минут у пользователя.

## User Story

Как пользователь с купленным пакетом минут, я хочу видеть спокойное информационное сообщение (а не пугающее предупреждение) о том, что дневной лимит исчерпан/почти исчерпан и теперь используются мои пакетные минуты, чтобы не беспокоиться о продолжении работы бота.

## Acceptance Criteria

- [ ] Если статус `"exhausted"` И у пользователя есть `package_minutes > 0` или `bonus_minutes > 0` — показывать мягкое информационное сообщение вместо `⛔ Дневной лимит исчерпан!`
- [ ] Если статус `"warning"` И у пользователя есть `package_minutes > 0` или `bonus_minutes > 0` — показывать мягкое информационное сообщение вместо `⚠️ Дневной лимит почти исчерпан!`
- [ ] Если у пользователя нет дополнительных минут (пакеты + бонусы = 0) — поведение не меняется (текущие пугающие предупреждения остаются)
- [ ] Мягкое сообщение при `"exhausted"` должно явно указывать, что дневной лимит исчерпан и используются пакетные/бонусные минуты
- [ ] Мягкое сообщение при `"warning"` должно явно указывать, что дневной лимит почти исчерпан и доступны пакетные/бонусные минуты
- [ ] Экран баланса (`build_balance_text_and_markup`) показывается в обоих случаях (при наличии и отсутствии пакетов)

## Dependencies

- `src/services/transcription_orchestrator.py` — основное место изменений (блок проверки лимитов после транскрипции)
- `src/services/payments/base.py` — `UserBalance.package_minutes`, `UserBalance.bonus_minutes`
- `src/services/billing_service.py` — `get_limit_status()`, `get_user_balance()`

## Out of Scope

- Изменение логики `get_limit_status()` в `billing_service.py` — метод остаётся неизменным
- Изменение пороговых значений (80% threshold, daily_limit) — не меняем
- Отображение предупреждений в других местах (кроме post-transcription блока в orchestrator)

В orchestrator уже есть код, который получает `balance` после `get_limit_status()`. Нужно лишь добавить проверку `has_extra_minutes = balance.package_minutes > 0 or balance.bonus_minutes > 0` и разветвить логику сообщений. Никакие дополнительные DB-запросы не нужны — `balance` уже содержит нужные поля.
