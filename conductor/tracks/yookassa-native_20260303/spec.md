# Specification: YooKassa Native Telegram Payments Integration

**Track ID:** yookassa-native_20260303
**Type:** Feature
**Created:** 2026-03-03
**Status:** Draft

## Summary

Заменить прямую интеграцию с YooKassa API (shop_id + secret_key + webhooks) на нативный Telegram Payments API с `provider_token`. Оплата картой будет работать так же, как Telegram Stars — через `sendInvoice` → `pre_checkout_query` → `successful_payment`, только с `currency="RUB"`.

## Context

В рамках треков billing-system_20260302, billing-review-fixes_20260302, billing-ux-fixes_20260302 была реализована полная биллинговая система. Telegram Stars работает через нативный Telegram Payments API. Для ЮКассы был написан `YooKassaProvider` с прямой интеграцией через API (shop_id, secret_key, return_url, webhooks), но он не подключён — кнопки карт показывают заглушку.

Пользователь получил `YOOKASSA_PROVIDER_TOKEN` через BotFather и хочет использовать нативный подход Telegram Payments, который проще, безопаснее и не требует webhook-сервера.

## User Story

As a пользователь бота, I want to оплачивать подписки и пакеты минут банковской картой через нативный интерфейс Telegram so that я не покидаю приложение и процесс оплаты прост и привычен.

## Acceptance Criteria

- [ ] Оплата картой через нативный Telegram Payments работает (подписки + пакеты минут)
- [ ] `YOOKASSA_PROVIDER_TOKEN` — единственная переменная окружения для ЮКассы (вместо shop_id/secret_key/return_url)
- [ ] Старый `YooKassaProvider` с прямым API удалён/заменён
- [ ] Кнопки "Оплата картой" реально работают (не заглушки)
- [ ] Тесты обновлены и проходят

## Dependencies

- Биллинговая система из billing-system_20260302 (реализована)
- Telegram Stars провайдер (работает, используется как образец)
- `YOOKASSA_PROVIDER_TOKEN` получен через BotFather

## Out of Scope

- Рекуррентные (автоматические) платежи через ЮКассу
- Webhook-сервер для прямого API ЮКассы
- Админ-панель для управления тарифами
- Чеки по 54-ФЗ (нативный Telegram Payments не поддерживает)
- Мультиязычность

## Technical Notes

- Новый `YooKassaProvider` будет аналогичен `TelegramStarsProvider`, но с `currency="RUB"` и `provider_token`
- Переиспользовать существующие `pre_checkout_query` и `successful_payment` хендлеры (или расширить)
- Удалить неиспользуемые env-переменные: `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`, `YOOKASSA_RETURN_URL`
- Убрать зависимость от пакета `yookassa` (если он был)
