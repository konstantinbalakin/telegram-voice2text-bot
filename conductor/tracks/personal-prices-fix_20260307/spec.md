# Specification: Проброс user_id для персональных цен и отображение expires_at в балансе

**Track ID:** personal-prices-fix_20260307
**Type:** Bug
**Created:** 2026-03-07
**Status:** Draft

## Summary

Персональные цены подписок и пакетов (записи с `user_id` в `subscription_prices` / `minute_packages`) не отображаются пользователям, потому что `billing_commands.py` не передаёт `user_id` через сервисный слой в репозитории. Дополнительно: в `/balance` не отображается срок годности минут (`expires_at`).

## Context

Система биллинга поддерживает персональные цены на уровне репозитория (`get_effective_prices`, `get_effective_packages`), но сервисный и UI-слои не пробрасывают `user_id`. Также `UserBalance` содержит только агрегированные суммы минут без информации о сроках годности.

## Problem Description

### Баг 1: Персональные цены не отображаются

**Цепочка вызовов (текущая)**:
1. `billing_commands.py:158` → `subscription_service.get_tier_prices(tier.id)` — **без user_id**
2. `subscription_service.py:80` → `subscription_repo.get_tier_prices(tier_id)` — **без user_id**
3. `billing_commands.py:196` → `payment_service.get_active_packages()` — **без user_id**
4. `payment_service.py:90` → `package_repo.get_active_packages()` — **без user_id**

Репозиторные методы `get_effective_prices(tier_id, user_id)` и `get_effective_packages(user_id)` принимают `user_id`, но он никогда не передаётся.

### Баг 2: expires_at не отображается в /balance

`_build_balance_text_and_markup` показывает только суммарные минуты:
```
🎁 Бонусные минуты: 45.0 мин
```

Нужно:
```
🎁 Бонусные минуты: 45.0 мин (до 15.04.2026)
```

Для записей с `expires_at != NULL` отображать дату рядом с суммой.
