# Specification: Billing UX Polish — Unified Screens, Descriptions & Bot Menu

**Track ID:** billing-ux-polish_20260305
**Type:** Feature
**Created:** 2026-03-05
**Status:** Draft

## Summary

Переработка UX биллинга: объединение экранов /balance и /buy в единый экран с инлайн-кнопками, редизайн каталогов подписок/пакетов с отдельным экраном описания для каждой опции, и внедрение полноценного меню команд бота через `set_my_commands`.

## Context

Система биллинга реализована (billing-system_20260302, yookassa-native_20260303), но требует UX-шлифовки перед запуском в продакшн. Текущие проблемы:
- Три отдельные команды (/balance, /subscribe, /buy) запутывают пользователя
- Каталоги показывают кнопки в два столбца (Stars/Рубли) — нет единой ценовой линейки
- Нет детального описания каждой опции перед оплатой
- Бот не регистрирует команды через Telegram Bot API — нет кнопки «Меню»

## User Stories

1. **Как пользователь бота**, я хочу удобно управлять подписками и покупками из одного экрана, чтобы не запоминать разные команды.
2. **Как пользователь бота**, я хочу видеть понятное описание каждой опции, чтобы осознанно выбирать тариф перед оплатой.

## Acceptance Criteria

### 1. Единый экран баланса и покупок
- [ ] Команды /balance и /buy показывают одинаковый экран: информация о балансе + инлайн-кнопки «Купить подписку» и «Купить пакеты минут»
- [ ] Команда /subscribe полностью удалена из кодовой базы
- [ ] Кнопка «Купить подписку» открывает подменю с вариантами подписок
- [ ] Кнопка «Купить пакеты минут» открывает подменю с вариантами пакетов
- [ ] В подменю есть кнопка «Назад» для возврата на основной экран

### 2. Редизайн каталогов
- [ ] Каталог подписок отображается в один столбец: название + цена в рублях + информация о скидке
- [ ] Каталог пакетов отображается в один столбец: название + цена в рублях + информация о скидке
- [ ] При нажатии на вариант подписки/пакета открывается отдельный экран с:
  - Названием опции
  - Подробным описанием услуги (из БД)
  - Двумя кнопками оплаты: 🇷🇺 💳 Карта РФ - <цена>₽)» и «🌟 Telegram Stars - <цена>⭐»
  - Кнопкой «Назад» к каталогу
- [ ] Описание (description) используется как подзаголовок в Telegram Payments invoice

### 3. Описания в БД
- [ ] Поле `description` добавлено в модель `SubscriptionPrice` (для каждого тарифа+период, например: «30 мин/день в течение недели»)
- [ ] Поле `description` добавлено в модель `MinutePackage`
- [ ] Миграция Alembic создана и применена
- [ ] Описания подтягиваются из БД при формировании экранов оплаты

### 4. Меню бота
- [ ] Команды регистрируются через `bot.set_my_commands()` при старте бота
- [ ] Все команды (кроме /start) с русскими названиями и эмодзи видны в кнопке «Меню»
- [ ] Список команд: /help, /balance, /buy + любые другие активные команды

## Technical Notes

### Затрагиваемые файлы
- `src/bot/billing_commands.py` — основная переработка экранов
- `src/bot/payment_callbacks.py` — новые callback handlers для промежуточных экранов
- `src/storage/models.py` — добавление description в SubscriptionPrice и MinutePackage
- `src/main.py` — регистрация set_my_commands, удаление /subscribe handler
- Alembic migration — новое поле description

### Callback data structure (новая)
- `billing:subscriptions` — открыть каталог подписок
- `billing:packages` — открыть каталог пакетов
- `billing:sub_detail:{tier_id}:{period}` — экран описания подписки
- `billing:pkg_detail:{package_id}` — экран описания пакета
- `billing:back_main` — назад к основному экрану баланса
- Существующие `sub_stars:`, `sub_card:`, `pkg_stars:`, `pkg_card:` — сохраняются для оплаты
