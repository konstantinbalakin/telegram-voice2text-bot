## Motivation

PR-ревью выявило:
- Биллинг **неработоспособен** из-за закрытой сессии БД (use-after-free)
- Перепутаны Telegram ID / Database ID — все billing-запросы ищут по неверному ключу
- Синхронные вызовы YooKassa SDK блокируют event loop
- Hardcoded email в чеке — нарушение 54-ФЗ
- Отсутствует верификация webhook YooKassa — риск поддельных платежей
- Нет error handling в billing pipeline — ошибка биллинга роняет транскрипцию
- Строковые литералы вместо enum — инварианты только в комментариях
- Purchase не помечается completed после оплаты
- Пробелы в тестовом покрытии edge cases
