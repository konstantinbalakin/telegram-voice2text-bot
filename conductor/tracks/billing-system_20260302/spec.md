# Specification: Billing System (Tarification)

**Track ID:** billing-system_20260302
**Type:** Feature
**Created:** 2026-03-02
**Status:** Draft

## Summary

Система тарификации для Telegram Voice2Text Bot: дневная квота бесплатных минут (10 мин/день), подписочная модель (Pro: 30 мин/день), пакеты минут (бессрочные), welcome-бонус (60 мин), оплата через Telegram Stars и ЮKassa.

## Context

Бот работает в полностью бесплатном режиме. Расходы на OpenAI Whisper API ~300₽/день и растут экспоненциально. ~14% пользователей генерируют ~60% расходов через длинные аудио. Себестоимость 1 минуты — 0.67₽. Необходима монетизация для устойчивости проекта.

Полный аналитический документ: `docs/research/monetization/2026-02-28-monetization-model.md`

## User Story

As a **владелец бота**, I want to **ввести систему тарификации с дневной квотой, подписками и пакетами минут** so that **покрыть расходы на API и обеспечить устойчивость проекта, сохранив базовый бесплатный доступ**.

As a **пользователь бота**, I want to **получать 10 бесплатных минут в день и иметь возможность докупить минуты** so that **продолжать использовать бот без перерывов**.

## Technical Notes

### Архитектура хранения тарифов

Все тарифные настройки хранятся в БД, не в ENV/config. Единственная ENV-переменная — `BILLING_ENABLED` (флаг включения платного режима). Система условий (общие + индивидуальные) использует таблицу `billing_conditions` с `valid_from`/`valid_to` и приоритетом индивидуальных над общими.

### Новые таблицы БД

- `billing_conditions` — общие и индивидуальные условия (дневной лимит, welcome-бонус и т.д.)
- `subscription_tiers` — уровни подписки (Pro, и т.д.)
- `subscription_prices` — цены (уровень × период)
- `user_subscriptions` — активные подписки пользователей
- `minute_packages` — каталог пакетов минут
- `user_minute_balances` — балансы минут (бонусные + пакетные)
- `daily_usage` — дневное использование минут
- `purchases` — история покупок (пакеты + подписки)
- `deduction_log` — детальный лог списания минут по источникам

### Ключевые сервисы

- `BillingService` — ядро: проверка лимитов, списание, балансы
- `SubscriptionService` — управление подписками (создание, продление, отмена)
- `PaymentService` — абстракция над платёжными провайдерами
- `TelegramStarsProvider` — интеграция с Bot Payments API
- `YooKassaProvider` — интеграция с ЮKassa API

### Интеграция с существующим кодом

- `TranscriptionOrchestrator.process_transcription()` — добавить проверку лимита перед транскрипцией
- `QueueManager` — списание минут последовательно через очередь
- `BotHandlers` — новые команды + обновлённый welcome
- `main.py` — DI для новых сервисов
